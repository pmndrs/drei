Pre-requisite: wrap into a `FaceLandmarker` provider:

### Basic usage

```jsx
<FaceLandmarker>
  <FaceControls />
</FaceLandmarker>
```

### With video source

```jsx
<FaceControls videoTexture={{ src: videoUrl }} />
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| camera | `THREE.Camera` | - | The camera to be controlled |
| videoTexture | `VideoTextureProps` | `{ start: true }` | VideoTexture or WebcamVideoTexture options |
| manualDetect | `boolean` | `false` | Disable the automatic face-detection => you should provide `faceLandmarkerResult` yourself in this case |
| faceLandmarkerResult | `FaceLandmarkerResult` | - | FaceLandmarker result |
| manualUpdate | `boolean` | `false` | Disable the rAF camera position/rotation update |
| makeDefault | `boolean` | - | Reference this FaceControls instance as state's `controls` |
| smoothTime | `number` | `0.25` | Approximate time to reach the target. A smaller value will reach the target faster. |
| offset | `boolean` | `true` | Apply position offset extracted from `facialTransformationMatrix` |
| offsetScalar | `number` | `80` | Offset sensitivity factor, less is more sensible |
| eyes | `boolean` | `false` | Enable eye-tracking |
| eyesAsOrigin | `boolean` | `true` | Force Facemesh's `origin` to be the middle of the 2 eyes |
| depth | `number` | `0.15` | Constant depth of the Facemesh |
| debug | `boolean` | `false` | Enable debug mode |
| facemesh | `FacemeshProps` | - | Facemesh options, default: undefined |
